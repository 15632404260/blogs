Git的简单介绍

Created2018-01-15|Git
|Post View:

Gitee 提供免费的 Git 仓库，还集成了代码质量检测、项目演示等功能。对于团队协作开发，Gitee 还提供了项目管理、代码托管、文档管理的服务，5 人以下小团队免费。

接下来我们学习一下如何使用 Gitee。

Git 的工作流程。

一般工作流程如下：

  • 克隆 Git 资源作为工作目录。

  • 在克隆的资源上添加或修改文件。

  • 如果其他人修改了，你可以更新资源。

  • 在提交前查看修改。

  • 提交修改。

  • 在修改完成后，如果发现错误，可以撤回提交并再次修改并提交。

首先登录Gitee

https://gitee.com/

新建仓库


git常用命令

将存储库的内容克隆到本地

git clone 链接地址

首先,作为 git 的基础配置，作用是告诉 git 你是谁，你输入的信息将出现在你创建的提交中。然后初始化项目

git config - -global user.name “你的名字或昵称”
git config - -global user.email “你的邮箱”
git init

将所有内容添加到暂存区 .代表所有

git add .

存储库记录更改

git commit -m “更改说明”

将仓库和服务器连接

git remote add origin 链接地址

删除远程地址

git remote rm origin

将本地仓库提交到远程仓库

git push origin master

查看仓库当前的状态，显示有变更的文件。

git status

查看提交日志

git log

代码回退

git reset –hard 提交ID

回退完需要提交到线上 但是只需要提交头部

git push origin HEAD –force

// force强制提交,注意：不能跳分支回退

Git 分支

什么是分支

分支是为了将修改记录的整体流程分叉保存。分叉后的分支不受其他分支的影响，所以在同一个数据库里可以同时进行多个修改。
简单理解：可以理解为复仇者联盟里面的多元宇宙，各个宇宙之间互不影响，到某一时间点会合并

好处

为了不受其他开发人员的影响，可以在主分支上建立自己专用的分支。
完成工作后，将自己分支上的修改合并到主分支。因为每一次提交的历史记录都会被保存，所以当发生问题时，定位和修改造成问题的提交就容易多了。

操作分支的简单命令

查看所有分支

git branch -a

查看云端分支

git branch -r

拉取云端代码or分支

git pull

创建分支

git branch 分支名

切换分支

git checkout -b 或者git switch -c

合并某分支到当前分支

git merge 分支名 或者 git rebase 分支名

merge可以保留所有提交，方便回滚，但不方便查看提交日志

rebase 只保留主历史，线性历史，便于查看，但每次都会把平行的历史给抹掉

删除分支

git branch -d
