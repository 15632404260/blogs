映射是什么？

映射 mappings 相当于原来由 elasticsearch 自动帮我们定义表结构。现在由我们自己来定义，旨在创建索引 的时候，有更多的定制内容，更加的贴合业务场景。

elasticsearch 中的映射用来定义一个文档及其包含的字段如何存储和索引的过程。我们可以使用映射来定义：

  • 哪些字符串应该被视为全文字段。
  • 哪些字段包含数字、日期或者地理位置。
  • 定义日期的格式。
  • 自定义的规则，用来投投动态添加字段的映射。

映射类型

每个索引都有一个映射类型，字决定了文档将如何被索引。

映射类型有：

  • 元字段（mate-fields）: 元字段用于自定义如何处理该当关联的元数据，如该当的 _index 、 _type 、 _id 和 _source 字段。
  • 字段和属性（field or properties）: 映射类型包含与文档相关的字段 或者属性的列表。

字段的数据类型

简单类型：文本（text）、关键字（keyword）、日期（data）、整型（long）、双精度（double）、布尔（boolean）或者 IP

支持 json 的层次结构性质的类型，对象或者嵌套

特殊类型：geo_point、peo_shape 或者 completion.

为了不同的目的，以不同的方式索引相同的字段 通常是有用的。例如，字符串字段 可以作来全文搜索的文本字段进行索引，也可以作为排序或聚合的关键字字段进行索引。或者，可以使用标准分析器、英文分析器和法语分析器索引字符串字段。

  • 启动容器成功后，可以访问一下浏览器: http://localhost:9200

结语：

  • 用过 haystack 和 whoosh 的我深深感到利用 ES 的方便和好用，因为 whoosh 他还要把数据库取出的结构化数据转换为非结构化数据存入 txt 文本中。但是当数据量大的时候呢，读文本为什么不读数据库。而 ES 他是面相文档的搜索引擎，他读出数据是在内存中读取，相对来说比 MYsql 要快很多。但是 ES 不能支持大量的写操作，因为他是不支持事务的。这也是 ES 的一大缺点，虽然他的读取速度加快了，但是比较消耗内存，性能较差的话的确没有 SQL 来的实际
